The LLM-Native OS

A typing-first OS where the LLM is the kernel, context is memory, and agents are apps. Minimize friction. Maximize flow.

View Repository

Architecture

Nerva lets you express intent directly through language, with the LLM routing your commands to the right tools and agents.

Core Concepts

  • LLM as kernel: The central orchestrator.
  • Context as memory: Active working memory buffers.
  • Tools as drivers: Controlled system capabilities.

UX Principles

  • Single input line: Intent via natural language.
  • Transcript-as-state: History is the UI.
  • Keyboard-first: No mouse required.

Quickstart

Get up and running with Node.js (v18+) and Python 3.10+.

Install

git clone https://github.com/SaiMani-Ritish/Nerva.git
cd Nerva
pnpm install
python -m venv venv
pip install -r requirements.txt

Run Shell

pnpm dev
nerva run shell

First Task

Try typing: "Index notes, suggest study plan, create tasks"

Configuration

Nerva is designed to be hackable. Config files are located in config/.

Models

Switch between local Llama models or Cloud providers like OpenAI/Anthropic seamlessly in models.yaml.

Tools

Enable/Disable filesystem access, web retrieval, and rate limits in tools.yaml.
